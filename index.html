BUKU PANDUAN PENGGUNAAN

Sistem Monitoring Survei (SMS) - Badan Pusat Statistik

1. PENGANTAR

Apa itu Sistem Monitoring Survei (SMS)?

Sistem Monitoring Survei (SMS) adalah sistem informasi berbasis web yang dikembangkan khusus untuk pemantauan progres kegiatan survei di lingkungan Badan Pusat Statistik (BPS). Sistem ini dirancang untuk mendukung koordinasi dan pemantauan survei dari tingkat pusat hingga satuan kerja daerah.

Tujuan Sistem

  • Centralized Management: Monitoring semua kegiatan survei dalam satu platform terpadu
  • Hierarchical Coordination: Mendukung koordinasi antar tingkat organisasi BPS
  • Real-time Monitoring: Memantau progress kegiatan survei secara otomatis
  • Standardized Process: Menstandarkan proses pelaksanaan survei
  • Audit Trail: Mencatat semua aktivitas untuk akuntabilitas

Struktur Organisasi yang Didukung

SMS dirancang untuk mendukung hierarki BPS:

1. BPS Pusat (Kode: 0000)

  • Membuat master kegiatan survei
  • Mengalokasikan kegiatan ke daerah
  • Monitoring pelaksanaan nasional

2. BPS Provinsi (Kode: XX00)

  • Mengelola kegiatan di tingkat provinsi
  • Mengkoordinasikan kabupaten/kota
  • Melaporkan ke pusat

3. BPS Kabupaten/Kota (Kode: XXXX)

  • Melaksanakan survei di lapangan
  • Pelaporan ke provinsi dan pusat

2. MEMULAI PENGGUNAAN SISTEM

Akses Sistem

  • URL Sistem: https://sms-fe-dev.student.stis.ac.id
  • Browser yang Didukung: Chrome, Firefox, Safari, Edge (versi terbaru)
  • Persyaratan: Koneksi internet stabil

Login ke Sistem

  1. Masuk ke Halaman Login
    Buka browser dan akses URL sistem. Anda akan diarahkan ke halaman login
  2. Memasukkan Kredensial
    • Email: Masukkan email yang telah terdaftar
    • Password: Masukkan password Anda
    • Klik tombol "Login"
  3. Setelah Login Berhasil
    Sistem akan menampilkan dashboard sesuai dengan peran Anda dan memberikan token autentikasi yang berlaku 24 jam
Lupa Password?
Jika lupa password, hubungi administrator sistem di satuan kerja Anda atau gunakan fitur reset password (jika tersedia).

3. PANDUAN BERDASARKAN PERAN

SUPERADMIN

Akses Penuh Sistem

  • Mengelola semua pengguna dan satuan kerja
  • Membuat dan mengelola role pengguna
  • Monitoring aktivitas di seluruh sistem
  • Konfigurasi sistem dan parameter

Menu Utama: Dashboard Super Admin, Manajemen Pengguna Global, Manajemen Satuan Kerja, Manajemen Role dan Hak Akses, Activity Logs Global, Konfigurasi Sistem

ADMIN PUSAT & OPERATOR PUSAT

Mengelola Tingkat Pusat

  • Membuat master kegiatan survei
  • Mengalokasikan kegiatan ke provinsi/satker
  • Monitoring pelaksanaan nasional
  • Mengelola direktorat dan deputinya

Workflow Utama:

  1. Membuat Master Kegiatan
    Buka menu "Master Kegiatan" → Klik "Tambah Kegiatan Baru" → Isi detail kegiatan → Assign ke direktorat → Simpan
  2. Mengalokasikan Kegiatan
    Pilih kegiatan → Klik "Assign ke Satker" → Pilih satker tujuan → Konfirmasi alokasi

ADMIN PROVINSI & OPERATOR PROVINSI

Mengelola Tingkat Provinsi

  • Menerima kegiatan dari pusat
  • Mengalokasikan ke kabupaten/kota
  • Monitoring provinsi
  • Koordinasi dengan satker

Workflow Utama:

  1. Menerima Kegiatan dari Pusat
    Cek menu "Kegiatan Provinsi" → Review kegiatan yang di-assign → Terima atau minta klarifikasi
  2. Mengalokasikan ke Kab/Kota
    Pilih kegiatan → Tentukan kabupaten/kota → Assign kegiatan → Monitor pelaksanaan

ADMIN SATKER & OPERATOR SATKER

Mengelola Tingkat Satker

  • Melaksanakan kegiatan survei
  • Mengelola tahapan survei
  • Upload dokumen dan laporan
  • Pelaporan progress

Workflow Utama:

  1. Melihat Kegiatan yang Ditugaskan
    Buka "Kegiatan Saya" → Lihat daftar kegiatan → Cek deadline dan requirements
  2. Melaksanakan Tahapan Survei
    Pilih kegiatan → Ikuti 8 tahap GSBPM → Update status subtahap → Upload dokumen

4. FITUR UTAMA SISTEM

1. Manajemen Kegiatan

Membuat Kegiatan Baru (Pusat Only)

  1. Klik menu "Master Kegiatan" → "Tambah Baru"
  2. Isi form dengan lengkap:
    • Nama kegiatan
    • Kode kegiatan (unik)
    • Deskripsi
    • Tanggal mulai dan selesai
    • Budget (jika ada)
    • Direktorat penanggung jawab
  3. Klik "Simpan"

2. Assignment Kegiatan

Assignment ke Multiple Satker

Pilih kegiatan master → Klik "Assign ke Satker" → Pilih satker dari daftar → Centang multiple satker → Klik "Assign"

Assignment Berdasarkan Provinsi

Pilih kegiatan master → Klik "Assign ke Provinsi" → Pilih provinsi → Sistem otomatis mengalokasikan ke semua satker di provinsi

3. Manajemen User

Menambah User Baru

  1. Klik menu "Users" → "Tambah User"
  2. Isi data user: Nama, Email, Password, Satuan kerja, Role
  3. Klik "Simpan"

Aktivasi/Deaktivasi User

Cari user → Klik toggle "Active/Inactive" → Konfirmasi perubahan

5. PENGELOLAAN TAHAPAN SURVEI

SMS menggunakan standar Generic Statistical Business Process Model (GSBPM) dengan 8 tahap utama:

Tahap 1: Specify Needs

6 Subtahap:

  1. Identifikasi kebutuhan informasi
  2. Konsultasi dan konfirmasi kebutuhan
  3. Penetapan tujuan output
  4. Identifikasi konsep dan definisi
  5. Cek ketersediaan data
  6. Persiapan proposal bisnis

Tahap 2: Design

6 Subtahap:

  1. Desain output
  2. Desain deskripsi variabel
  3. Desain kuesioner
  4. Desain frame dan sample
  5. Desain proses dan analisis
  6. Desain alur kerja

Tahap 3: Build

7 Subtahap:

  1. Menggunakan kembali atau membangun kuesioner
  2. Menggunakan kembali atau membangun komponen proses dan analisis
  3. Menggunakan kembali atau membangun komponen diseminasi
  4. Konfigurasi alur kerja
  5. Uji sistem produksi
  6. Uji proses bisnis statistik
  7. Finalisasi sistem produksi

Tahap 4: Collect

4 Subtahap:

  1. Buat frame dan pilih sample
  2. Persiapan pengumpulan data
  3. Pengumpulan data
  4. Finalisasi hasil pengumpulan data

Tahap 5: Process

8 Subtahap:

  1. Integrasi data
  2. Klasifikasi dan pengkodean
  3. Review dan validasi
  4. Edit dan imputasi data
  5. Membuat variabel turunan
  6. Penghitungan bobot
  7. Penghitungan agregasi
  8. Finalisasi data

Tahap 6: Analyse

5 Subtahap:

  1. Menyiapkan draft output
  2. Validasi output
  3. Interpretasi dan menjelaskan output
  4. Menerapkan kontrol diseminasi
  5. Finalisasi output

Tahap 7: Disseminate

5 Subtahap:

  1. Perbarui sistem output
  2. Membuat produk diseminasi
  3. Mengelola rilis produk diseminasi
  4. Promosi produk diseminasi
  5. Mengelola bantuan konsumen diseminasi

Tahap 8: Evaluate

3 Subtahap:

  1. Mengumpulkan masukan evaluasi
  2. Melakukan evaluasi
  3. Penyetujuan rencana tindakan

Cara Menggunakan Tahapan:

  1. Mengakses Tahapan
    Pilih kegiatan dari "Kegiatan Saya" → Klik "Kelola Tahapan" → Sistem menampilkan 8 tahap dengan progress
  2. Mengupdate Status Subtahap
    Klik subtahap → Centang "Selesai" → Isi tanggal realisasi → Tambahkan keterangan → Simpan
  3. Upload Dokumen
    Di setiap tahap → Klik "Upload Dokumen" → Pilih file → Tambahkan deskripsi → Upload
  4. Melihat Progress
    Progress ditampilkan dalam persentase, status, visualisasi timeline, dan comparison planned vs actual

Status Tahapan:

  • Belum Mulai: Tahap belum dikerjakan
  • Sedang Berjalan: Tahap dalam proses pengerjaan
  • Selesai Tepat Waktu: Selesai sesuai atau lebih cepat dari jadwal
  • Selesai Terlambat: Selesai tetapi melewati deadline
  • Terlambat: Melewati deadline tapi belum selesai

7. TROUBLESHOOTING

Masalah Login

"Email atau Password Salah"

  • Pastikan email dan password benar
  • Cek caps lock
  • Coba reset password
  • Hubungi admin jika masih bermasalah

"Token Expired"

  • Login ulang ke sistem
  • Token berlaku 24 jam
  • Jika sering expired, hubungi admin

Masalah Upload File

"File Gagal Upload"

  • Cek ukuran file (max sesuai setting sistem)
  • Pastikan format file didukung
  • Cek koneksi internet
  • Coba upload ulang

"Access Denied"

  • Pastikan Anda memiliki hak akses
  • Cek apakah file sedang digunakan aplikasi lain
  • Hubungi admin untuk konfirmasi hak akses

Masalah Performa

"Sistem Lambat"

  • Cek koneksi internet
  • Tutup tab browser yang tidak perlu
  • Clear cache browser
  • Coba refresh halaman

"Data Tidak Tersimpan"

  • Pastikan koneksi internet stabil
  • Cek mandatory field sudah diisi
  • Coba simpan kembali
  • Hubungi admin jika persisten

8. FAQ (Frequently Asked Questions)

Umum

Q: Berapa lama token login berlaku?

A: Token login berlaku selama 24 jam. Setelah itu perlu login ulang.

Q: Apakah bisa mengakses sistem dari mobile?

A: Ya, sistem responsive dan dapat diakses dari mobile browser.

Q: Bagaimana cara mengubah password?

A: Buka menu Profile → Change Password, masukkan password lama dan baru.

Kegiatan

Q: Siapa yang bisa membuat kegiatan baru?

A: Hanya ADMIN_PUSAT dan OPERATOR_PUSAT yang bisa membuat master kegiatan.

Q: Bagaimana cara melihat kegiatan yang ditugaskan ke saya?

A: Buka menu "Kegiatan Saya" di dashboard.

Q: Apakah bisa mengubah kegiatan yang sudah di-assign?

A: Perubahan terbatas, hubungi admin untuk perubahan major.

Tahapan

Q: Apakah harus mengikuti urutan tahapan?

A: Ya, tahapan mengikuti standar GSBPM dan sebaiknya berurutan.

Q: Bagaimana jika tahapan terlambat?

A: Sistem akan memberikan notifikasi. Update status dan berikan keterangan keterlambatan.

Q: Bisakah upload multiple file dalam satu tahap?

A: Ya, bisa upload beberapa file untuk setiap tahap.

Hak Akses

Q: Bagaimana cara request akses tambahan?

A: Hubungi admin satker atau level yang lebih tinggi.

Q: Kenapa tidak bisa melihat data satker lain?

A: Akses dibatasi berdasarkan hierarchical scope untuk keamanan data.

Teknis

Q: Browser apa yang direkomendasikan?

A: Chrome, Firefox, Safari, atau Edge versi terbaru.

Q: Apakah data otomatis tersimpan?

A: Tidak, pastikan selalu klik "Simpan" setelah input data.

Q: Bagaimana cara backup data?

A: Sistem melakukan backup otomatis. Untuk export data pribadi, gunakan fitur export di setiap modul.
